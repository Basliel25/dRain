# dRain

DRain is a streaming, thread-safe log template extraction engine. It implements the Drain algorithm (He et al., ICWS 2017)[https://jiemingzhu.github.io/pub/pjhe_icws2017.pdf] as a Rust library that links against the `trafilo` C framework for event transport.

**Depends on:** [trafilo](https://github.com/Basliel25/trafilo) for event transport

## What It Does

Drain discovers structural templates from raw log lines from a log stream, without domain knowledge. 

##### The rough data flow
1. Raw log line arrives from Trafilo via FFI callback
2. `preprocess()` to replace IPs, numbers, hex with `<*>` and other repeatings with an placeholder
3. `tokenize()`the raw input removing white space and other delimeters
4. Template and then travesr tree, root -> length bucket -> first-token bucket -> leaf cluster
5. On finding matching leaf it return a `template_id`
6. If no match is found it creates new template, return new `template_id`
7. If similar (above threshold) merge templates via wildcard promotion

**Tree** in the first version is plannned to be a fixed depth tree (4 or 6) for benchmarking and testing, 
Roughly a tree generated by dRain would look like 

```
Root
|
+-- Length=3 bucket
| |
| +-- FirstToken="sshd" bucket
| | |
| | +-- Leaf: "sshd Failed password"
| | +-- Leaf: "sshd Accepted publickey"
| |
| +-- FirstToken="cron" bucket
| |
| +-- Leaf: "cron finished job"
|
+-- Length=5 bucket
|
+-- ...
```

This tree will be to generate a pattern to infere future semantics of log fils.

##### Trafilo Integration
Trafilo (C) calls dRain (Rust) through a single `extern "C"` entry point

dRain will the expose to trafilo:
```
pub extern "C" fn drain_parse(
    raw_line: *const c_char,
    template_id_out: *mut u64,
    params_out: *mut *mut c_char,
    params_len: *mut i32,
) -> i32;
```
Trafilo will also be the owner of the raw line
